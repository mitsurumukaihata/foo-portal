# ブリヂストンA表 Excelのクセ・取り込み仕様メモ

最終更新: 2026/04/25
担当: 向畑 充 / Claude Code

毎月・四半期ごとに届くA表を取り込む前に **必ずこのメモを読み直し**、
今月分のExcelが過去のクセを踏襲しているか確認すること。

---

## 📂 ファイル配置ルール

```
~/Desktop/売上明細/タイヤメーカー価格表/
  ★【YYYY年N月～verX.XX】夏タイヤ価格表_BRIDGESTONE.xlsx
  ★【YYYY年N月～verX.XX】冬タイヤ価格表_BRIDGESTONE.xlsx
  (他メーカー分は別ファイル)
```

ファイル名の `YYYY年N月～` 部分から **価格有効開始日** を読み取る。
取り込み時は必ず D1 の `最終更新日` を書き換える。

## 🧪 サイズ表記のクセ

### 1. LT (Light Truck) 規格 vs P規格
- `LT215/75R15` と `P215/75R15` は **別物**
- 現状は `215/75R15` で統一してしまっている → 将来的にはプレフィックス保持が必要

### 2. 扁平比の特殊値
- `00` = 扁平比なし(古いLT規格 600R14 等) → サイズ `600R14` に
- `99` = 規格番号の99シリーズ? → 現状 `扁平なし` として扱っている
- 0 と空欄は同じ扱い

### 3. LI/SS(ロードインデックス/速度記号)の扱い
- 商品名の先頭にLI+SS付く (`089Y`, `*95Y`, `073S`)
- D1には **格納していない** (パターン・サイズとの組合せで特定可能なため)
- 必要なら `商品名称` から正規表現で再抽出可能

### 4. XL(エクストラロード) 扱い
- K列(XL) の値 | 取り込みルール |
  - `XL` → サイズ末尾に ` XL` 付与
  - `XLPR` / `PR` / `RF` / `LLPR` → **現状無視**
  - 空欄 → 通常規格

## 💰 価格欄の扱い

- A表列(V)に値がある行 = 販売対象商品 (4,357件 / 2026年2月)
- A表列が **0 / 空** の行 = 92件
  - 新発売前?
  - 廃番予定?
  - 要相談?
- **推奨**: 価格空行も D1 に保存し、`価格 IS NULL` として「要確認」表示
- 現実装もこの方針

## 🔁 取り込みスクリプトの実行手順

```bash
# 新A表Excelが届いたら:
cd foo-portal
node scripts/import-bs-a-table-excel.mjs "C:/Users/.../★【YYYY年M月～verX.XX】夏タイヤ価格表_BRIDGESTONE.xlsx"
```

処理内容:
1. `価格リスト` シート全行読み込み
2. グループコード→カテゴリ変換
3. ブランドコード→ブランド名変換
4. 商品名称からパターン抽出(正規表現)
5. `A表` シートからマーク記号409件抽出
6. D1 の BS 行を全削除
7. 新データ 4,411行(程度) を INSERT
8. wrangler 経由で反映(約10秒)

## 🔧 取り込み精度の現状(2026/04/25)

✅ **完全取り込み**:
- 商品コード
- サイズ (小さな誤差: LT/P 区別なし)
- 価格
- 商品名称 (フルテキスト)
- RINC品名 (カタカナ名)
- ★②◇等のマーク

⚠️ **精度に幅あり**:
- `パターン` 列 → 正規表現抽出のため、一部で末尾ノイズが残る場合あり
  - 例: "GRV" と出るべきところに "235/50 R18 XL GRV" と残ることあり
  - 対処: 商品名称フル保存しているので、検索時は商品名称でも引ける

❌ **未対応**:
- LT/P 規格の区別
- XLPR/PR/RF サフィックスの明示
- 承認元記号(A5/M1/OE等)の意味付け表示
